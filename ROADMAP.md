## 🛠️ 완료된 세부 구현 사항 (Completed Tasks)

### 1. FZF 기반 대화형 TUI 전향 (`src/interactive.sh`)
* **구현 내용**: `Ctrl + G`를 눌렀을 때 백그라운드에서 직접 쉘 버퍼를 조작하던 구방식을 탈피하고, 독립적인 FZF 대화형 세션에서 입력을 받고 결과를 연동하도록 전환했습니다.
* **이점**: 터미널 화면 깨짐, 개행 꼬임, `command not found` 에러가 원천 예방됩니다.

### 2. 질의 히스토리 기록 및 선택창 (`~/.config/ctrlg/history.log`)
* **구현 내용**: 과거에 질문했던 한글/자연어 질의(Query) 목록만 파일에 저장하고 최신순으로 정렬하여 FZF 목록에 뿌려줍니다.
* **이점**: 이전 질의를 선택하면 즉시 해당 질문을 AI에 재전송하여 실시간 추천 명령어를 생성합니다.

### 3. Zsh Zle Invalidate (`zle -I`) 통합
* **구현 내용**: Zsh 위젯 실행 중 로그가 출력되는 시점에 `zle -I`를 선언하여 쉘 에디터를 일시 중지시키고, 로그가 끝난 후 새 라인에 프롬프트를 그리도록 동기화했습니다.
* **이점**: Zsh 환경에서 원래 작성 중이던 줄이 지워지지 않고 깔끔하게 보존됩니다.

## 📅 향후 개선 및 미완료 과제 (Future Backlog)

### 1. JSON 기반 설정 관리 고도화 (`config.json`)
* **목표**: 현재의 `config.env` 기반 설정을 `config.json` 포맷으로 구조화하고 `jq`를 사용하여 효율적으로 파싱 및 가동합니다.
* **작업 내용**:
  * `src/config_loader.sh` 모듈 생성
  * CLI에서 설정을 즉시 변경할 수 있는 `ctrlg config set key value` 서브명령어 구현

### 2. 다중 명령어 후보군 매핑 및 fzf 선택창 제공
* **목표**: AI 모델이 단일 결과만 추천하는 대신, 유사도나 목적에 따른 2~3개의 명령어 후보군을 리스트로 제공하고, 사용자가 그중 하나를 FZF로 선택하여 프롬프트에 주입하는 기능입니다.
* **작업 내용**:
  * 시스템 프롬프트 튜닝하여 JSON 또는 다중 라인 형태로 후보군 명령어 출력 유도
  * 생성된 후보군을 FZF 리스트로 렌더링하여 선택 적용 지원

### 3. 크로스플랫폼 확장 (Windows PowerShell 및 WSL)
* **목표**: Linux 환경을 넘어 Windows PowerShell(PSReadLine) 및 WSL 세션 간 RAG 캐시 싱크를 지원합니다.
* **작업 내용**:
  * 플랫폼 감지 모듈 (`src/platform.sh`) 작성
  * 파워쉘 콘솔 입력창 주입 및 Windows 클립보드 복사 API 연동
